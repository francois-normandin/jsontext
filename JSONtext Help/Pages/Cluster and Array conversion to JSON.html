Cluster conversion to/from JSON Objects or Arrays

LabVIEW clusters are ordered lists of named or unnamed elements (of mixed datatype).   JSON Arrays are ordered lists of unnamed items, while JSON Objects are unordered collections of uniquely-named items.  Thus, it is not a one-to-one mapping between a LabVIEW cluster and JSON types.

JSONtext, when converting a cluster to JSON, follows the following convention, based on the name of the first cluster element:

  • If the first element is unnamed (or has the exact name <JSON>), then the cluster is converted to a JSON Array.
  • If the first element is named then the cluster is converted to a JSON Object.

Only the first cluster element is used to make this decision.  If converted to a JSON Array, then any names of elements are ignored. If converted to a Object, then unnamed elements will have an empty name ("").  

Clusters must have unique names in order to produce a valid JSON Object.

Array conversion

LabVIEW arrays are ordered lists of a sigle datatype.   They can be converted to/from JSON Arrays of all one type.   To handle mixed-type JSON Arrays use a cluster with unnamed elements (can have <JSON> tags), as described above.

For support please see [CR] JSONtext.
